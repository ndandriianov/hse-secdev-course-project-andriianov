# Security Policy

## Аутентификация

- При ≥ 5 неуспешных логинов за 3 мин учётка блокируется на 5 мин
- Ответ унифицирован
- Причины не раскрываются


## Сессии

- Срок жизни access-токена ≤ 30 минут
- При смене пароля - отзыв всех сессий ≤ 5 мин


## Ошибки/ответы

- Ответы аутентификации и валидации унифицированы
- Cтатусы/коды без раскрытия внутренних причин


## Секреты

- Секреты загружаются только из .env


## Логи/аудит

- ≥ 80% критичных действий порождают событие audit{actor, action, resource, result, correlation_id) через ≤ 15 сек после операции.


## Приватность/ретеншн
- Храним только минимально-необходимые поля ПДн
- В АРI/логах маскирование
