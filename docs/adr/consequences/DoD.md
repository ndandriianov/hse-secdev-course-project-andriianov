Definition of Done (DoD)
- Общие
  - Изменения покрыты unit/integration тестами и проходят CI (lint + tests + security gates).
  - Документация обновлена (README/SECURITY/OpenAPI).
  - Назначен ответственный за поддержку (owner) на период rollout.

- ADR - ошибки (RFC 7807 / упрощённый формат)
  - Все HTTP-ошибки возвращают стандартизованный JSON с полями: код/статус/сообщение + correlation_id.
  - В публичных ответах нет стектрейсов или сырых внутренних данных.
  - correlation_id логируется и связывает ответ с серверными логами.
  - Автотест гарантирует отсутствие чувствительных полей в ответах.

- ADR - даты/деньги (UTC, Decimal, локаль)
  - API принимает/отдаёт datetime в ISO-8601 UTC (с Z); входы без TZ отклоняются.
  - Money поля используются как Decimal / numeric; в API - строка или {amount, currency}.
  - Сериализация/десериализация централизована; есть тесты round-trip.
  - DB поля для денег/timestamps проверены или миграции покрыты бэкапом и тестом.

- ADR - секреты (источник, маскирование, ротация)
  - .env.example присутствует; .env в .gitignore.
  - gitleaks/semgrep запускаются в pre-commit и CI; CI падает при обнаружении секретов.
  - Логи маскируют password/token/secret; есть тесты на маскирование.
  - Документирован базовый процесс ротации и владелец процесса назначен.
