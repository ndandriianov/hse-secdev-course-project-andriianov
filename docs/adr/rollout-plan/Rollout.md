rollout - шаги (стадии)
1. Подготовка (локально / feature-ветка)
   - Реализовать изменения, добавить тесты и документацию.
   - Добавить feature-flag для ключевых изменений (если возможно).

2. Staging
   - Запустить в staging; прогнать CI, контрактные и интеграционные тесты.
   - Прогнать миграции на копии БД, проверить бэкап/откат.
   - Проверить: ошибки в формате, парсинг дат/денег, маскирование логов, correlation_id в логах.

3. Rollout
   - Выкатить в prod
